O design foca na eficiência técnica e controle de dados locais, adequado para usuários que gerenciam modelos binários e indexação RAG. A interface prioriza a densidade de informação e a clareza funcional.

### 1. Color (Esquema de Cores)
- Fundo principal: Cinza escuro sólido (#121212) para conforto em uso prolongado e foco no conteúdo.
- Cor primária: Verde terminal (#00FF66) aplicado estritamente para indicar status de sistema (modelo ativo, TTS ligado, botão de envio).
- Cor de destaque: Laranja técnico (#FF6B00) exclusivo para exceções (erros de formato, pastas inacessíveis).
- Superfícies: Cinza chumbo (#1E1E1E) para diferenciar o Painel de Controle e os balões do sistema.
- Texto: Cinza claro (#E0E0E0) para leitura, garantindo alto contraste com o fundo. Proibido uso de gradientes.

### 2. Layout (Estrutura de Layout)
- Estrutura de tela dividida: Área de chat ocupando a coluna principal; Painel de Controle operando como uma barra lateral fixa ou gaveta sobreposta, permitindo ajuste de parâmetros sem perder o contexto da conversa.
- Área de chat: Alinhamento vertical rigoroso. Mensagens limitadas a 80% da largura da tela para otimizar a leitura.
- Painel de Controle: Organização em blocos modulares (Gestor, RAG, Parâmetros). Uso de grid denso para agrupar controles técnicos de forma compacta.
- Espaçamento: Sistema de grid estrito de 8px para criar hierarquia através do espaço em branco, sem linhas divisórias desnecessárias.

### 3. Elements (Elementos de Interface)
- Tipografia: Fonte monoespaçada para caminhos de pastas RAG, nomes de arquivos `.gguf` e valores numéricos; fonte sans-serif neutra para o texto do chat.
- Formas: Componentes com bordas retas ou raio mínimo (2px), transmitindo precisão técnica e utilitarismo.
- Controles: Sliders de Temperatura e Tokens exibem o valor numérico exato permanentemente ao lado do controle.
- Anexos: Representados por blocos retangulares contendo a extensão do arquivo em texto e o peso (ex: `documento.pdf | 2MB`), sem ilustrações complexas.
- Ícones: Vetores de linha fina (1.5px), estritamente funcionais (engrenagem, clipe de papel).

### 4. Motion (Design de Animação)
- Geração de resposta: O texto do modelo aparece progressivamente, refletindo o processamento do motor de inferência local.
- Feedback de interação: Mudanças de estado (ativar TTS, selecionar modelo) utilizam transições rápidas de opacidade (150ms).
- Navegação: Abertura do Painel de Controle via deslizamento horizontal (`transform: translateX`) com curva de aceleração `ease-out` (200ms).
- Uploads: Arquivos anexados surgem na interface através de um `fade-in` simples, confirmando a ação sem atrasar o fluxo do usuário.
